# 阿丽小红书排版 Skill (New Brutalism Style)

本 Skill 用于把一篇文章转成小红书图文卡片，采用 **New Brutalism（新粗野主义）** 视觉风格，强制执行「视觉高压」排版规范——**严禁留白**，单页空白面积不得超过 10%。

---

## 0. 执行前必读

**生成任何卡片之前，必须先读取 `references/reference.html`。**

这是本 Skill 的视觉锚点,是一份完整的 14 页真实作品。所有排版、配色、组件样式、视觉密度、多页节奏,以该文件为准。SKILL.md 里的规则是对这份示例的**文字补充**,不是替代——示例里没覆盖的细节,优先沿用示例的做法,而不是自由发挥。

**重要区分**:reference.html 里的**文字内容仅供视觉排版参考**,不得直接抄袭或复用。每次生成卡片时,内容必须完全来自用户当次提供的文章,严格保留用户原文的叙事、金句和个人化表达。

读取方式:直接打开 `references/reference.html` 全文,理解其 CSS 变量、组件结构、视觉密度、配色语义、多页节奏。

## 1. 风格定义：New Brutalism

### 视觉特征

- **极粗黑色边框**：`.canvas` 必须带 `14px solid #000`
- **强烈阴影偏移**：所有 `.section-box` 必须带 `6px 6px 0px #000`（或 `10px 10px 0px #000` 强调版）
- **网格底纹**：背景使用 `radial-gradient(#d1d5db 1px, transparent 1px)`，尺寸 `24px 24px`
- **高对比字重**：主要文本字重 900，副本 700，避免 400 以下

### 色彩逻辑（语义化，不可乱用）

| 颜色 | Hex | 语义 |
|------|------|------|
| 蓝色 | `#dbeafe` | 识别信号、逻辑起点、概念引入 |
| 黄色 | `#fef3c7` | 流程操作、迭代改动、过程中的动作 |
| 绿色 | `#d1fae5` | 成功结果、价值产出、正向示范 |
| 红色 | `#fee2e2` | 报错、错误示范、Manual/Error 对比 |
| 高亮 | `#fef08a` | 关键金句划重点、一句话结论 |

## 2. 视觉组件（每页必选至少一种）

每一页必须包含以下至少一种可视化组件，作为占页面 **50% 面积**的视觉锚点。组件的 HTML 结构参见 `references/reference.html`：

- **信号探测条（Frequency Bar）**：展示 3 次原则演进或触发频率
- **对比 VS 框**：左红（Manual/Error）vs 右绿（Skill/Success）
- **进度路径图（Progress Path）**：垂直或水平排列的步骤节点
- **核心矩阵（Core Matrix）**：四宫格、表格或平铺标签组
- **OS 窗口（OS Window）**：模拟带红绿灯按钮的系统窗口，增加 Vibe 感

---

## 3. 叙事与排版规范

### 100% 叙事保留

- **严禁**删减用户原文的背景故事、逻辑链路、比喻、个人化表达
- 个人化表达（如"我吃过亏""说了三个月烦了"）**必须**保留原词，不得改写成"根据经验"这类通用表达
- 深度文案必须封装在带阴影和边框的容器内，**严禁**直接平铺在背景上

### 字号红线

- **主标题**：44px–68px，Weight 900，行高 1.1
- **正文**：24px–28px，Weight 500/700，`word-break: break-all`
- **底部金句**：每页底部必须包含一个背景色显眼、字重 900 的重点短句（15–20 字），**强制单行**

### 分页逻辑

- **页数硬红线：全套卡片总数严禁超过 19 张**
- 同一维度内容必须合并在单页，通过高密度组件（Grid / Table）压缩
- 首页 = 封面 + 核心标题，末页 = 总结 + 互动 CTA

---

## 4. 自动化与交付

- **必选库**：`html2canvas`, `JSZip`（通过 CDN 引入）
- **导出逻辑**：循环捕捉所有 `.canvas`，每次捕捉间添加 300ms 滚动延迟，最终打包为 `小红书全套卡片.zip`
- **专属 Vibe Tag**：底部必须带有「大头阿丽 · Vibe Coding 系列」的标识

---

## 5. 易错点自检（生成后必查）

生成完成后，AI 必须逐条自查以下清单：

- [ ] 是否读取了 `references/reference.html`？
- [ ] 单页空白面积是否 ≤10%？
- [ ] 是否保留了所有个人化叙事（"我吃过亏""烦了"等）？
- [ ] 每页是否至少有一个可视化组件？
- [ ] 主标题字号是否在 44–68px 之间？
- [ ] 底部金句是否单行、字重 900？
- [ ] 颜色使用是否符合语义逻辑？（蓝 ≠ 警告，绿 = 成功）
- [ ] 右上角标签（Badge）是否遮挡了标题或内容？（不得遮挡）
- [ ] 总页数是否 ≤19 张？
- [ ] 是否包含 `html2canvas` + `JSZip` 的导出按钮？
- [ ] 是否包含「大头阿丽 · Vibe Coding 系列」的 Vibe Tag？
